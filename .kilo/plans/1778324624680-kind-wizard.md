# Plan: Telegram Group Preset Scraper

## Overview

Create a Python script that connects to specified Telegram groups, downloads `.txt` preset files shared in those groups, and outputs them in a format compatible with the existing Spooky2 extraction pipeline.

## Architecture

### API Choice: Telethon (Telegram Client API)

- **Why Telethon over Bot API**: Bots cannot freely read message history in groups (especially pre-existing messages). Telethon uses a user account to act as a client, allowing full access to group message history including file attachments.
- **Alternative considered**: Pyrogram — equally capable, but Telethon is more Pythonic and has better async support.
- The script will use a Telegram user account (phone login), not a bot.

### Directory Structure

```
scripts/
  telegram_scraper.py        # Main scraper script
  telegram_scraper_config.py # Auto-generated config (gitignored)
data/
  telegram/                  # Downloaded raw files organized by group
    <group_name>/
      <message_id>_<filename>.txt
```

## Implementation Steps

### 1. Setup & Dependencies

- Add `telethon` to project requirements (`scripts/requirements.txt` or top-level)
- Create `scripts/telegram_scraper.py` as the main entry point

### 2. Configuration

The script will accept configuration via:
- A config file (`scripts/telegram_scraper_config.py` — gitignored) containing:
  - `api_id` / `api_hash` (from https://my.telegram.org)
  - `phone_number` for login
  - `target_groups`: list of group usernames/IDs to scrape
  - `output_dir`: path to save downloaded files
- Command-line arguments (override config):
  - `--groups`: comma-separated list of group identifiers
  - `--output`: output directory (default: `data/telegram`)
  - `--since`: optional date filter (ISO format) to limit message range
  - `--limit`: max number of messages to process per group

### 3. Authentication Flow

```python
from telethon import TelegramClient

client = TelegramClient('session_name', api_id, api_hash)
client.start(phone=phone_number)  # Interactive: sends SMS code prompt
```

- First run will prompt for phone number and SMS/2FA code
- Session file persists login for subsequent runs (`session_name.session`)

### 4. Group Discovery & Resolution

- Accept group identifiers as: `@username`, numeric ID, or invite link
- Resolve each to an `InputPeer` via `client.get_entity()`
- Filter to only groups/channels (not individual users)

### 5. Message Iteration & File Download

For each target group:
1. Iterate through messages using `client.iter_messages(entity, limit=N)`
2. Filter messages that contain `.txt` file attachments:
   ```python
   if message.document and message.document.mime_type == 'text/plain':
   ```
3. Also capture messages containing preset-like text content (fallback):
   ```python
   if message.message and ('List2=' in message.message or 'PresetName=' in message.message):
   ```
4. Download each file: `await client.download_media(message, file=output_path)`
5. Also save message metadata (sender, date, message_id) as a sidecar `.meta.json`

### 6. Filename Convention

Downloaded files follow: `{message_id}_{original_filename}`
- Preserves uniqueness across groups
- Enables traceability back to the source message
- Directory structure: `data/telegram/<group_safe_name>/`

### 7. Rate Limiting & Error Handling

- Telethon handles Telegram's flood limits automatically (built-in delays)
- Implement retry logic (3 attempts) for transient failures
- Log skipped/failed messages to `scrape_errors.log`
- Handle `FloodWaitError` gracefully (Telethon does this by default)

### 8. Integration with Existing Pipeline

After download, the user can:
1. Manually review downloaded `.txt` files
2. Run existing extraction: `python3 scripts/extract_presets.py <preset_source_dir> --output data/presets`
3. The telegram-downloaded files can be placed alongside other preset source files

Optional: Add a `--extract` flag to the scraper that runs the extraction pipeline automatically after downloading.

### 9. Output Summary

After each run, print:
- Groups scraped
- Messages processed
- Files downloaded
- File sizes (total)
- Any errors encountered

## Security & Privacy

- **API credentials**: Stored in gitignored config file
- **Session files**: Gitignored (`.gitignore` updated)
- **Data usage**: Only downloads publicly shared files in groups the user has access to
- **No spam/noise**: Script is read-only, no messages sent

## Risks

1. **Telegram API bans**: Excessive scraping may trigger rate limits. Mitigation: built-in delays, configurable limits.
2. **Group access**: Script can only access groups the user account is a member of.
3. **File format inconsistency**: Some presets may be screenshots (images) rather than text files — out of scope for v1.
4. **API ID requirement**: User needs to register an app at my.telegram.org (free, instant).
